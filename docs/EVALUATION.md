# Evaluation status

Last updated: 2026-07-18 (Asia/Ho_Chi_Minh).

## What exists

The current reproducible evidence has four layers:

1. A four-case hero contract covering two distinct roots for the same surface target, safe
   abstention, and a strong-learner fast path.
2. A **disclosed 24-profile development suite** generated by an independent noisy response
   simulator. It contains six fraction-root, six ratio-root, six competing-root and six ready
   profiles. Slip and guess parameters range from 0.02 to 0.12 and every seed is visible in Git.
3. Deterministic safety and calibration contracts covering answer/method separation,
   misconception evidence from independent items, detect–verify–escalate dispositions, a finite
   teacher-time allocator, Brier score and expected calibration error (ECE).
4. `agent-eval-v1`: five frozen harness scenarios covering missing-evidence refusal, a
   general-explanation negative control, prompt-injection/tool-allowlist containment, mutation
   approval and task-level token/latency accounting.

The suite is synthetic. It measures deterministic contract behavior, not student learning,
curriculum validity, fairness or classroom impact.

## Current disclosed-suite result

| Measure | Result | Interpretation |
| --- | ---: | --- |
| Exact outcome | 19/24 (79.2%) | Five high-noise cases conservatively request more evidence |

| Best always-answer baseline | 12/24 (50.0%) | Lowest-mastery baseline; it cannot abstain or fast-path |
| Difference from best baseline | +29.2 pp | Development result only; not a held-out claim |
| Local diagnosis p95 gate | <300 ms | Enforced in tests; timing varies by machine |
| Misconception support | 2 distinct items required | One item remains a hypothesis needing verification |
| Teacher attention budget | 12/15 minutes selected | Exact deterministic allocation; 3 minutes remain |
| Synthetic Brier/ECE gates | both <0.25 over 144 KC points | Disclosed synthetic ground truth, not real students |

The baseline comparison covers all outcomes. Restricting evaluation only to known root cases would
reward an always-answer baseline for guessing and would erase the product requirement to abstain on
ambiguous evidence and advance already-mastered learners.

## Agent eval v1

`tests/eval/agent-safety-v1.test.ts` runs without a remote model and treats the model/provider as
untrusted. The runtime must replace unsupported learner claims with an explicit lack-of-evidence
response without blocking a general teaching explanation, reject a prompt-injected tool outside
the session allowlist, and keep a mutation from executing without teacher approval. A deterministic
telemetry case records input, output, cached tokens, TTFT and total latency.

Managed ChatGPT subscription use does not expose an attributable currency cost per task, so the
eval records monetary cost as unavailable instead of inventing a USD estimate. Provider/API routes
may add a real cost field later only when their billing contract supplies a defensible calculation.

## Reproduce

Use Node 24.18.0 LTS, then run:

```powershell
npm ci
npm test
npm run eval
```

The definitions are in `tests/eval/development-suite.ts`; the independent simulator, named
baselines, calibration report and deterministic contracts are adjacent in `tests/eval/`.

## Still blocked on human/independent ownership

- Six held-out labels must be frozen by someone who does not tune the inference implementation.
- Curriculum edges, question wording, answer keys and intervention actions remain `UNREVIEWED`
  until a named teacher/mentor records approval.
- The disclosed suite gives every method the same complete synthetic history. A later online-budget
  evaluation must test accuracy and abstention after each additional question.
- No result may be described as SOTA, causal improvement or real-world learning gain.
